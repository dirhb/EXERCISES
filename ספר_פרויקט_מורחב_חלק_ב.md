# פרק 4: מימוש הפרויקט – צד שרת

## 4.1 ארכיטקטורת השרת

פרויקט **JobWebService** הוא ליבת המערכת. הוא מספק ממשק REST API המאורגן לפי בקרים, כאשר כל בקר אחראי על תחום מוגדר היטב של הלוגיקה העסקית. הבקרים אינם כותבים SQL ישירות – הם מאצילים את העבודה לשכבת ה-Repository דרך יחידת העבודה (`LibraryUOW`).

### תרשים UML פשוט של שכבת השרת

```
ControllerBase (ASP.NET Core)
├── GuestController          ← REST endpoint: /api/Guest/...
├── UserController           ← REST endpoint: /api/User/...
├── EmployerController       ← REST endpoint: /api/Employer/...
├── AdminController          ← REST endpoint: /api/Admin/...
└── ChatController           ← REST endpoint: /api/Chat/...

LibraryUOW (Unit of Work)
├── DBHelperOledb            ← OleDbConnection, OleDbCommand
├── ModelCreators            ← Factory מחלקות
└── Repositories (1 per Entity):
    ├── UserRepository       ← IRepository<User>
    ├── JobRepository        ← IRepository<Job>
    ├── JobApplicationRepository
    ├── ChatMessageRepository
    ├── GenreRepository
    ├── CountryRepository
    ├── ReviewRepository
    ├── NotificationRepository
    ├── UserTypeRepository
    ├── EducationRepository
    └── EducationTypeRepository
```

## 4.2 שכבת ה-Model

שכבת ה-Model מיושמת בפרויקט **JobModels** שהוא Class Library. כל מחלקה במודל מייצגת ישות בודדת מעולם הדומיין ומכילה Property אחת לכל שדה בטבלת בסיס הנתונים המקבילה. הסוגים כולם מוגדרים כ-Nullable (`?`) כדי לאפשר גמישות מקסימלית בעת דה-סריאליזציה (Deserialization) של JSON.

### מחלקת `User.cs` – מחלקת המודל המרכזית


**הערות על הורשה ועיצוב:**
כל מחלקות ה-Model בפרויקט זה הן מחלקות POCO (Plain Old C# Objects) ללא הורשה הדדית. הן מכילות רק Properties ואין בהן לוגיקה עסקית. עיצוב זה מתאים לתבנית Repository Pattern שבה הלוגיקה מרוכזת במחלקות ה-Repository ולא במודלים עצמם.

## 4.3 שכבת ה-ORM וגישה לנתונים

### מחלקת `DBHelperOledb.cs` – שכבת הגישה לנתונים

מחלקה זו היא ליבת שכבת הנתונים. היא עוטפת (Wraps) את `OleDbConnection` ו-`OleDbCommand` ומספקת ממשק מופשט (Abstracted Interface) ל-Repository Classes.

**האובייקטים הפרטיים:**
```csharp
public class DBHelperOledb : IDatabaseHelper
{
    OleDbConnection oleDbConnection; // מנהל את החיבור למסד הנתונים
    OleDbCommand oleDbCommand;       // שולח פקודות SQL ומחזיר תוצאות
    OleDbTransaction oleDbTransaction; // מנהל Transactions (מאוחד)
```

**Constructor ואתחול:**
```csharp
public DBHelperOledb()
{
    this.oleDbConnection = new OleDbConnection();
    this.oleDbCommand = new OleDbCommand();

    string databasePath = ResolveDatabasePath();
    this.oleDbConnection.ConnectionString = 
        $@"Provider=Microsoft.ACE.OLEDB.12.0;Data Source={databasePath};Persist Security Info=True";
    
    this.oleDbCommand.Connection = this.oleDbConnection;
    EnsureChatMessagesTableExists(); // בדיקה ויצירה אוטומטית של טבלת הצ'אט
}
```

**Connection String מנותחת:**
- `Provider=Microsoft.ACE.OLEDB.12.0` – ספק ה-OleDb ל-Access 2007 ומעלה (קבצי `.accdb`).
- `Data Source={databasePath}` – נתיב מלא לקובץ `.accdb`.
- `Persist Security Info=True` – שמירת מידע אבטחה בחיבור.

**Method לאיתור בסיס הנתונים:**
```csharp
private static string ResolveDatabasePath()
{
    const string databaseFileName = "JobFindUltimate.accdb";
    
    // סריקת ספריות בהיררכיה עולה עד למציאת הקובץ
    foreach (string startPath in new[] { Directory.GetCurrentDirectory(), AppContext.BaseDirectory })
    {
        DirectoryInfo? directory = new DirectoryInfo(startPath);
        while (directory != null)
        {
            string candidate = Path.Combine(directory.FullName, "App_Data", databaseFileName);
            if (File.Exists(candidate)) return candidate;

            candidate = Path.Combine(directory.FullName, "JobWebService", "App_Data", databaseFileName);
            if (File.Exists(candidate)) return candidate;

            directory = directory.Parent;
        }
    }
    return Path.Combine(Directory.GetCurrentDirectory(), "App_Data", databaseFileName);
}
```

Method זה פותר בעיה נפוצה: כאשר הפרויקט מופעל מסביבת Visual Studio, תיקיית העבודה הנוכחית עשויה להיות שונה מ-`AppContext.BaseDirectory`. הסריקה ההיררכית מבטיחה שהקובץ יימצא ללא קשר לאופן ההפעלה.

**Methods מרכזיות:**

```csharp
// פתיחת חיבור
public void OpenConnection() => this.oleDbConnection.Open();

// סגירת חיבור
public void CloseConnection() => this.oleDbConnection.Close();

// הוספת פרמטר לפקודה - מנגנון ההגנה מ-SQL Injection
public void AddParameters(string name, object value)
{
    this.oleDbCommand.Parameters.Add(new OleDbParameter(name, value ?? DBNull.Value));
}

// שליפת נתונים - מחזיר IDataReader לקריאת שורה-שורה
public IDataReader Read(string sql)
{
    this.oleDbCommand.CommandText = sql;
    IDataReader datareader = this.oleDbCommand.ExecuteReader();
    oleDbCommand.Parameters.Clear(); // ניקוי פרמטרים לאחר ביצוע
    return datareader;
}

// קריאת ערך יחיד (COUNT, MAX, וכד')
public object ReadValue(string sql)
{
    this.oleDbCommand.CommandText = sql;
    try
    {
        return this.oleDbCommand.ExecuteScalar();
    }
    finally
    {
        this.oleDbCommand.Parameters.Clear(); // תמיד מנקה גם בחריגה
    }
}

// ביצוע שינוי (INSERT/UPDATE/DELETE) - מחזיר מספר שורות שהושפעו
private int ChangeDb(string sql)
{
    this.oleDbCommand.CommandText = sql;
    try
    {
        return this.oleDbCommand.ExecuteNonQuery();
    }
    finally
    {
        this.oleDbCommand.Parameters.Clear();
    }
}

public int Insert(string sql) => ChangeDb(sql);
public int Update(string sql) => ChangeDb(sql);
public int Delete(string sql) => ChangeDb(sql);
public int Create(string sql) => ChangeDb(sql);
```

**Method לאתחול טבלת הצ'אט:**
```csharp
private void EnsureChatMessagesTableExists()
{
    bool tableExists = false;
    try
    {
        this.oleDbConnection.Open();
        using (var cmd = this.oleDbConnection.CreateCommand())
        {
            // מנסה לשלוף שורה אחת - אם הטבלה לא קיימת, תיזרק חריגה
            cmd.CommandText = "SELECT TOP 1 MessageID FROM ChatMessages";
            using (var reader = cmd.ExecuteReader()) { }
            tableExists = true;
        }
    }
    catch { tableExists = false; } // טבלה לא קיימת
    finally { this.oleDbConnection.Close(); }

    if (!tableExists)
    {
        try
        {
            this.oleDbConnection.Open();
            using (var cmd = this.oleDbConnection.CreateCommand())
            {
                // יצירת הטבלה עם כל העמודות הנחוצות
                cmd.CommandText = 
                    "CREATE TABLE ChatMessages " +
                    "(MessageID COUNTER PRIMARY KEY, " +
                    "SenderID VARCHAR(100), " +
                    "ReceiverID VARCHAR(100), " +
                    "MessageText MEMO, " +
                    "SentAt VARCHAR(50))";
                cmd.ExecuteNonQuery();
            }
        }
        finally { this.oleDbConnection.Close(); }
    }
}
```

### ממשק `IRepository<T>` – ממשק מחסן הנתונים הגנרי

```csharp
public interface IRepository<T>
{
    bool Insert(T model);
    T Read(object id);
    List<T> ReadAll();
    bool Update(T model);
    bool Delete(int id);
    object ReadValue();
}
```

ממשק זה מגדיר פעולות CRUD גנריות (Create, Read, ReadAll, Update, Delete) שכל Repository חייב לממש. הגנריות (`<T>`) מאפשרת לאותו ממשק לשמש עבור כל ישות: `IRepository<User>`, `IRepository<Job>`, `IRepository<ChatMessage>` וכדומה.

### מחלקת `Repository.cs` – בסיס מופשט

```csharp
public class Repository
{
    protected DBHelperOledb helperOleDb;
    protected ModelCreators modelCreators;

    public Repository(DBHelperOledb helperOleDb, ModelCreators modelCreators)
    {
        this.helperOleDb = helperOleDb;
        this.modelCreators = modelCreators;
    }
}
```

מחלקת הבסיס `Repository` מכילה את שני האובייקטים המשותפים לכל ה-Repositories: `DBHelperOledb` (לביצוע שאילתות) ו-`ModelCreators` (ליצירת אובייקטי מודל מ-`IDataReader`). כל Repository ספציפי יורש ממחלקה זו.

### מחלקת `UserRepository.cs` – ניתוח מלא

```csharp
public class UserRepository : Repository, IRepository<User>
{
    // Constructor: מעביר את ה-Dependencies לבסיס
    public UserRepository(DBHelperOledb helperOleDb, ModelCreators modelcreators) 
        : base(helperOleDb, modelcreators) { }

    // בדיקת ייחודיות שם משתמש
    public bool IsAvailableUserName(string userName)
    {
        string sql = "SELECT 1 FROM [User] WHERE UserName=@UserName";
        this.helperOleDb.AddParameters("@UserName", userName);
        using (IDataReader dr = this.helperOleDb.Read(sql))
        {
            return !dr.Read(); // true אם לא נמצאה שורה = שם זמין
        }
    }

    // שליפת סיסמה לפי שם משתמש (לצורך אימות)
    public String GetPasswordByUserName(string userName)
    {
        string sql = "SELECT Password FROM [User] WHERE UserName=@UserName";
        this.helperOleDb.AddParameters("@UserName", userName);
        using (IDataReader dr = this.helperOleDb.Read(sql))
        {
            if (dr == null || !dr.Read()) return null;
            return Convert.ToString(dr["Password"]);
        }
    }

    // INSERT משתמש חדש
    public bool Insert(User model)
    {
        string sql = "INSERT INTO [User] " +
                     "(FirstName,LastName,UserName,[Password],Country," +
                     "PhoneNum,CreationDate,UserTypeID,Email) " +
                     "VALUES(@FirstName,@LastName,@UserName,@Password," +
                     "@Country,@PhoneNum,@CreationDate,@UserTypeID,@Email)";
        
        this.helperOleDb.AddParameters("@FirstName", model.FirstName);
        this.helperOleDb.AddParameters("@LastName", model.LastName);
        this.helperOleDb.AddParameters("@UserName", model.UserName);
        this.helperOleDb.AddParameters("@Password", model.Password);
        this.helperOleDb.AddParameters("@Country", model.Country);
        this.helperOleDb.AddParameters("@PhoneNum", model.PhoneNum);
        this.helperOleDb.AddParameters("@CreationDate", model.CreationDate);
        this.helperOleDb.AddParameters("@UserTypeID", 
            model.UserTypeID.HasValue ? model.UserTypeID.Value.ToString() : null);
        this.helperOleDb.AddParameters("@Email", model.Email);
        
        return this.helperOleDb.Create(sql) > 0;
    }

    // UPDATE שכר עובד (ייעודי לבקשת מעסיק)
    public bool UpdateSalary(string userId, decimal salary)
    {
        string sql = "UPDATE [User] SET Salary=@Salary WHERE UserID=@UserID";
        this.helperOleDb.AddParameters("@Salary", salary.ToString());
        this.helperOleDb.AddParameters("@UserID", userId);
        return this.helperOleDb.Update(sql) > 0;
    }

    // UPDATE קורות חיים (URL לקובץ PDF)
    public bool UpdateResume(string userId, string resumeText)
    {
        string sql = "UPDATE [User] SET ResumeText=@ResumeText WHERE UserID=@UserID";
        this.helperOleDb.AddParameters("@ResumeText", resumeText);
        this.helperOleDb.AddParameters("@UserID", userId);
        return this.helperOleDb.Update(sql) > 0;
    }

    // SELECT משתמש לפי אישורים (דוא"ל + סיסמה)
    public User GetByCredentials(string email, string password)
    {
        string sql = "SELECT [User].*, Country.CountryName FROM [User] " +
                     "LEFT JOIN Country ON [User].Country = CStr(Country.CountryID) " +
                     "WHERE [User].Email=@Email AND [User].Password=@Password";
        this.helperOleDb.AddParameters("Email", email);
        this.helperOleDb.AddParameters("Password", password);
        using (IDataReader dataReader = this.helperOleDb.Read(sql))
        {
            if (dataReader == null || !dataReader.Read()) return null;
            return this.modelCreators.UserCreator.CreateModel(dataReader);
        }
    }
}
```

```csharp
public class JobRepository : Repository, IRepository<Job>
{
    // INSERT משרה חדשה

    {
        string sql = @"INSERT INTO Job(JobTitle,JobDescription,JobType,

                        @JobFilter,@EmployerID,@CountryID,@GenreID)";

        this.helperOleDb.AddParameters("@JobFilter", model.JobFilter);
        this.helperOleDb.AddParameters("@EmployerID", model.EmployerID);
        this.helperOleDb.AddParameters("@CountryID", model.CountryID);

    public List<Job> ReadAll()
    {

                     "LEFT JOIN Genre ON Job.GenreID = Genre.GenreID";
```

**קוד מלא של `GetJobCatalog` עם סינון:**
```csharp
[HttpGet]
public JobCatalog GetJobCatalog(string? location, string? employmentType, 
    string? salaryRange, string? experienceLevel, string? flexibility)
{
    try
    {
        this.libraryUOW.HelperOledb.OpenConnection();
        List<Job> jobs = this.libraryUOW.JobRepository.ReadAll();

        // סינון לפי מיקום (CountryID)
        if (!string.IsNullOrWhiteSpace(location))
            jobs = jobs.Where(x => x.CountryID == location).ToList();

        // סינון לפי סוג משרה (Full-time, Part-time, וכד')
        if (!string.IsNullOrWhiteSpace(employmentType))
            jobs = jobs.Where(x => x.JobType == employmentType).ToList();

        // סינון לפי רמת ניסיון (חיפוש ב-JobFilter)
        if (!string.IsNullOrWhiteSpace(experienceLevel))
            jobs = jobs.Where(x => (x.JobFilter ?? "")
                .Contains(experienceLevel, StringComparison.OrdinalIgnoreCase)).ToList();

        // סינון לפי גמישות (Remote, Hybrid וכד')
        if (!string.IsNullOrWhiteSpace(flexibility))
            jobs = jobs.Where(x => (x.JobFilter ?? "")
                .Contains(flexibility, StringComparison.OrdinalIgnoreCase)).ToList();

        // סינון לפי טווח שכר (חיפוש ב-JobFilter)
        if (!string.IsNullOrWhiteSpace(salaryRange))
            jobs = jobs.Where(x => (x.JobFilter ?? "")
```

## 4.5 אבטחה והגנה מפני SQL Injection

### הסבר מעמיק על SQL Injection

SQL Injection היא פגיעות אבטחה שמאפשרת לתוקף להזריק קוד SQL זדוני לתוך שאילתה לגיטימית. הנה דוגמה להמחשה:

**שאילתה פגיעה (Vulnerable):**
```csharp
// NEVER DO THIS:
string sql = "SELECT * FROM [User] WHERE Email='" + email + "' AND Password='" + password + "'";
```

אם תוקף מזין כסיסמה את המחרוזת: `' OR '1'='1' --`
השאילתה הופכת ל:
```sql
SELECT * FROM [User] WHERE Email='user@example.com' AND Password='' OR '1'='1' --'
```
הביטוי `OR '1'='1'` תמיד נכון, ו-`--` מבטל את שאר השאילתה. התוקף נכנס ללא סיסמה.

**השאילתה המאובטחת בפרויקט:**
```csharp
// הפתרון הנכון - Parameterized Query:
string sql = "SELECT * FROM [User] WHERE Email=@Email AND Password=@Password";
this.helperOleDb.AddParameters("Email", email);
this.helperOleDb.AddParameters("Password", password);
```

כאשר `OleDbCommand` מקבל פרמטר מסוג `OleDbParameter`, הוא מבצע Escaping אוטומטי של כל תווים מיוחדים בתוך הערך. הערך `' OR '1'='1' --` יתפרש כמחרוזת טקסט פשוטה ולא כפקודת SQL.

## 4.6 Unit of Work – יחידת העבודה

### תבנית UoW ויתרונותיה

מחלקת `LibraryUOW` מיישמת את תבנית **Unit of Work (יחידת עבודה)**. תבנית זו מרכזת את כל ה-Repositories תחת מרכיב אחד, ומבטיחה שכולם חולקים אותו חיבור למסד הנתונים (`DBHelperOledb` יחיד).

**יתרון מרכזי:** כל ה-Repositories מקבלים את **אותו** אובייקט `DBHelperOledb`. כך, אם שני Repositories שונים מבצעים פעולות בתוך אותו בקר, הם עושים זאת דרך אותו חיבור, ואפשר לנהל Transaction אחד לכל פעולות הבקר.

**מנגנון Lazy Initialization:**
```csharp
public JobRepository JobRepository
{
    get
    {
        // יצירת Repository רק בפעם הראשונה שנגישים אליו (Lazy)
        if (jobRepository == null)
            jobRepository = new JobRepository(helperOledb, modelcreators);
        return jobRepository;
    }
}
```
מנגנון ה-Lazy Initialization מבטיח שאובייקטי Repository נוצרים רק כאשר יש בהם צורך, חוסך זיכרון ומניעת יצירת אובייקטים מיותרים.

## 5.2 מחלקת ApiClient – תקשורת א-סינכרונית עם השרת

מחלקת `ApiClient<T>` היא אחד הרכיבים החשובים ביותר בצד הלקוח. היא מאפשרת לבקרים לתקשר עם שרת ה-API בצורה נוחה, א-סינכרונית ומבוטחת טיפוסים (Type-Safe).

### אובייקטים ועיצוב

```csharp
public class ApiClient<T>
{
    // HttpClient סטטי (Shared) - יחיד לכל האפליקציה
    HttpClient httpClient = LibraryHttpClient.Instance;
    // UriBuilder לבניית כתובת ה-URL דינמית
    UriBuilder uriBuilder = new UriBuilder();
```

**Singleton של `HttpClient`:** שימוש ב-`LibraryHttpClient.Instance` חשוב מאוד. יצירת `HttpClient` חדש בכל בקשה היא שגיאה ידועה ב-.NET שגורמת ל-Socket Exhaustion (מחסור ב-Sockets). ה-`Singleton` מבטיח שאובייקט אחד ויחיד מנוהל את כל החיבורים.

### בניית ה-URL דינמית

```csharp
// Properties לבניית ה-URL:
public string Scheme { set { this.uriBuilder.Scheme = value; } }
public string Host { set { this.uriBuilder.Host = value; } }
public int Port { set { this.uriBuilder.Port = value; } }
public string Path { set { this.uriBuilder.Path = value; } }

// הוספת פרמטר Query String:
public void AddParameter(string key, string value)
{
    if (this.uriBuilder.Query == string.Empty)
        this.uriBuilder.Query += "?";
    else
        this.uriBuilder.Query += "&";
    this.uriBuilder.Query += $"{key}={value}";
}
```

**מנגנון שימוש ב-EmployeeController:**
```csharp
private ApiClient<T> BuildClient<T>(string controller, string action)
{
    ApiClient<T> client = new ApiClient<T>();
    client.Scheme = "http";       // פרוטוקול
    client.Host = "localhost";    // כתובת השרת
    client.Port = 5015;           // פורט השרת
    client.Path = $"api/{controller}/{action}"; // נתיב ה-Endpoint
    return client;
}
```

### קריאת GET א-סינכרונית

```csharp
public async Task<T> GetAsync()
{
    using (HttpRequestMessage httpRequest = new HttpRequestMessage())
    {
        httpRequest.Method = HttpMethod.Get;
        httpRequest.RequestUri = this.uriBuilder.Uri;
        
        // await: מחזיר Thread ל-Pool בזמן ההמתנה
        using (HttpResponseMessage httpResponse = 
            await this.httpClient.SendAsync(httpRequest))
        {
            if (httpResponse.IsSuccessStatusCode == true)
            {
                string result = await httpResponse.Content.ReadAsStringAsync();
                
                // דה-סריאליזציה של JSON לאובייקט C#
                JsonSerializerOptions jsonSerializerOptions = new JsonSerializerOptions();
                jsonSerializerOptions.PropertyNameCaseInsensitive = true; // לא case-sensitive
                T model = JsonSerializer.Deserialize<T>(result, jsonSerializerOptions);
                return model;
            }
            return default(T); // null לאובייקטים, 0 למספרים
        }
    }
}
```

**הסבר `PropertyNameCaseInsensitive = true`:**
שרת ה-API מחזיר JSON עם properties באותיות קטנות: `{"jobID": "1", "jobTitle": "..."}`.
מחלקת ה-Model ב-C# מגדירה properties באותיות גדולות: `JobID`, `JobTitle`.
האפשרות `PropertyNameCaseInsensitive = true` מאפשרת ל-`JsonSerializer` להתאים ביניהם ללא כשל.

### קריאת POST א-סינכרונית עם גוף JSON

```csharp
public async Task<bool> PostAsync(T model)
{
    using (HttpRequestMessage httpRequest = new HttpRequestMessage())
    {
        httpRequest.Method = HttpMethod.Post;
        httpRequest.RequestUri = this.uriBuilder.Uri;
        
        // סריאליזציה של האובייקט לפורמט JSON
        string json = JsonSerializer.Serialize<T>(model);
        StringContent content = new StringContent(json);
        httpRequest.Content = content;
        
        using (HttpResponseMessage responseMessage = 
            await this.httpClient.SendAsync(httpRequest))
        {
            return responseMessage.IsSuccessStatusCode == true;
        }
    }
}
```

### קריאת POST עם העלאת קובץ (Multipart)

```csharp
public async Task<bool> PostAsync(T model, Stream file)
{
    using (HttpRequestMessage httpRequest = new HttpRequestMessage())
    {
        httpRequest.Method = HttpMethod.Post;
        httpRequest.RequestUri = this.uriBuilder.Uri;
        
        // MultipartFormDataContent: מאפשר שליחת גוף מורכב (JSON + קובץ)
        MultipartFormDataContent multipartFormData = new MultipartFormDataContent();
        
        string json = JsonSerializer.Serialize<T>(model);
        StringContent modelContent = new StringContent(json);
        multipartFormData.Add(modelContent, "model"); // חלק JSON
        
        StreamContent streamContent = new StreamContent(file);
        multipartFormData.Add(streamContent, "file", "file"); // חלק קובץ
        
        httpRequest.Content = multipartFormData;
        
        using (HttpResponseMessage responseMessage = 
            await this.httpClient.SendAsync(httpRequest))
        {
            return responseMessage.IsSuccessStatusCode == true;
        }
    }
}
```

## 5.3 ניהול Session ו-SessionHelper

### Session ב-ASP.NET Core

Session הוא מנגנון לשמירת מידע על המשתמש בין בקשות HTTP. HTTP הוא פרוטוקול Stateless (חסר מצב) – כל בקשה עצמאית ואינה "זוכרת" את הבקשה הקודמת. Session פותר זאת על ידי:
1. יצירת מזהה Session ייחודי (Session ID).
2. שמירת המזהה ב-Cookie בדפדפן.
3. שמירת הנתונים (Data) על השרת.
4. בכל בקשה, הדפדפן שולח את ה-Cookie, והשרת מאתר את ה-Session המתאים.

```csharp
public static class SessionHelper
{
    // קבועים למפתחות ה-Session - מונעים שגיאות כתיב
    private const string USER_ID   = "UserID";
    private const string USER_TYPE = "UserTypeID";
    private const string USER_NAME = "UserName";
    private const string FULL_NAME = "FullName";

    // שמירת פרטי משתמש לאחר התחברות מוצלחת
    public static void SetUser(ISession session, string userId, int userTypeId, 
                                string userName, string fullName)
    {
        session.SetString(USER_ID,   userId);
        session.SetString(USER_TYPE, userTypeId.ToString());
        session.SetString(USER_NAME, userName);
        session.SetString(FULL_NAME, fullName);
    }

    // קריאת מזהה משתמש
    public static string? GetUserID(ISession session) => session.GetString(USER_ID);

    // קריאת סוג משתמש כ-Integer
    public static int GetUserTypeID(ISession session)
    {
        int.TryParse(session.GetString(USER_TYPE), out int result);
        return result; // מחזיר 0 אם לא מחובר
    }

    // בדיקה אם משתמש מחובר
    public static bool IsLoggedIn(ISession session) =>
        !string.IsNullOrEmpty(session.GetString(USER_ID));

    // בדיקות תפקיד (Role Checks)
    // UserTypeID: 1=Guest, 2=Employee, 3=Employer, 4=Admin
    public static bool IsEmployee(ISession session) => GetUserTypeID(session) == 2;
    public static bool IsEmployer(ISession session) => GetUserTypeID(session) == 3;
    public static bool IsAdmin(ISession session)    => GetUserTypeID(session) == 4;

    // ניתוק (Logout) - מחיקת כל נתוני Session
    public static void ClearUser(ISession session) => session.Clear();
}
```

**יתרונות `SessionHelper`:**
- **מניעת שגיאות כתיב:** השימוש בקבועים (`USER_ID = "UserID"`) מבטיח עקביות בכל הפרויקט. בלי קבועים, שגיאה כמו `session.GetString("userId")` (u קטנה) ביחד עם `session.SetString("UserID", ...)` (U גדולה) לא תמצא ולא תוצג שגיאת קומפילציה.
- **קריאות קוד:** `SessionHelper.IsEmployee(session)` הרבה יותר קריאה מ-`int.TryParse(session.GetString("UserTypeID"), out int t) && t == 2`.
- **נקודת שינוי יחידה:** אם רוצים לשנות את שם מפתח ה-Session, מספיק לשנות אותו בקבוע אחד.

```csharp
public async Task<IActionResult> JobCatalog(
    string? search, string? employerId, string? genreId, 
    string? jobType, bool showActiveOnly = false, int page = 1)
{
    if (!IsAuthorized()) return RedirectToAction("Home", "Guest");

    const int pageSize = 16; // 16 כרטיסי משרה לדף

    // שליפת כל המשרות מה-API
    ApiClient<List<Job>> client = BuildClient<List<Job>>("Guest", "GetAllJobs");
    List<Job> allJobs = await client.GetAsync() ?? new List<Job>();

    // בניית רשימת מעסיקים ייחודיים לסרגל הסינון
    List<string> employers = allJobs
        .Select(job => job.EmployerID ?? string.Empty)
        .Where(id => !string.IsNullOrWhiteSpace(id))
        .Distinct(StringComparer.OrdinalIgnoreCase)
        .OrderBy(id => id)
        .ToList();

    // שליפת תחומים (Genres) דינמית מה-API
    ApiClient<List<Genre>> genreClient = BuildClient<List<Genre>>("Guest", "GetAllGenres");
    List<Genre> genres = await genreClient.GetAsync() ?? new List<Genre>();

    // אלגוריתם סינון: LINQ Pipeline
    IEnumerable<Job> filteredJobs = allJobs;

    // סינון 1: חיפוש חופשי על פני 7 שדות
    if (!string.IsNullOrWhiteSpace(search))
        filteredJobs = filteredJobs.Where(job => MatchesJobSearch(job, search));

    // סינון 2: לפי מעסיק
    if (!string.IsNullOrWhiteSpace(employerId))
        filteredJobs = filteredJobs.Where(job => 
            string.Equals(job.EmployerID, employerId, StringComparison.OrdinalIgnoreCase));

    // סינון 3: לפי תחום - גמיש (על GenreID או GenreTitle)
    if (!string.IsNullOrWhiteSpace(genreId))
        filteredJobs = filteredJobs.Where(job =>
            string.Equals((job.GenreID ?? "").Trim(), genreId.Trim(), StringComparison.OrdinalIgnoreCase) ||
            string.Equals((job.GenreTitle ?? "").Trim(), genreId.Trim(), StringComparison.OrdinalIgnoreCase));

    // סינון 4: לפי סוג משרה
    if (!string.IsNullOrWhiteSpace(jobType))
        filteredJobs = filteredJobs.Where(job => 
            string.Equals((job.JobType ?? "").Trim(), jobType.Trim(), StringComparison.OrdinalIgnoreCase));

    // סינון 5: הצגת פעילות בלבד
    if (showActiveOnly)
        filteredJobs = filteredJobs.Where(job => job.JobStatus == true);

    List<Job> results = filteredJobs.ToList();

    // חישוב Paging:
    int totalPages = Math.Max(1, (int)Math.Ceiling(results.Count / (double)pageSize));
    page = Math.Clamp(page, 1, totalPages); // הגבלת page לתחום תקין

    // העברת נתונים לתצוגה דרך ViewBag:
    ViewBag.Jobs = results.Skip((page - 1) * pageSize).Take(pageSize).ToList();
    ViewBag.TotalJobs = results.Count;
    ViewBag.Page = page;
    ViewBag.TotalPages = totalPages;
    ViewBag.Genres = genres;
    ViewBag.Employers = employers;
    // ... שאר ViewBag values

    return View();
}

// פונקציית חיפוש על 7 שדות:
private static bool MatchesJobSearch(Job job, string search)
{
    return (job.JobTitle ?? string.Empty).Contains(search, StringComparison.OrdinalIgnoreCase)
        || (job.JobDescription ?? string.Empty).Contains(search, StringComparison.OrdinalIgnoreCase)
        || (job.JobType ?? string.Empty).Contains(search, StringComparison.OrdinalIgnoreCase)
        || (job.JobFilter ?? string.Empty).Contains(search, StringComparison.OrdinalIgnoreCase)
        || (job.EmployerID ?? string.Empty).Contains(search, StringComparison.OrdinalIgnoreCase)
        || (job.CountryID ?? string.Empty).Contains(search, StringComparison.OrdinalIgnoreCase)
        || (job.GenreID ?? string.Empty).Contains(search, StringComparison.OrdinalIgnoreCase);
}
```

```csharp
[HttpPost]
public async Task<IActionResult> ApplyToJob(int jobId, IFormFile? resumeFile, string? returnUrl)
{
    if (!IsAuthorized()) return RedirectToAction("Home", "Guest");
    
    string userId = SessionHelper.GetUserID(HttpContext.Session)!;
    string? resumeUrl = null;

    if (resumeFile != null && resumeFile.Length > 0)
    {
        // ולידציה Server-Side - בדיקת סיומת
        var extension = Path.GetExtension(resumeFile.FileName).ToLower();
        if (extension != ".pdf")
        {
            TempData["Error"] = "Only PDF files are allowed.";
            return RedirectToReturnUrl(returnUrl);
        }

        // בדיקת גודל
        if (resumeFile.Length > 5 * 1024 * 1024)
        {
            TempData["Error"] = "File size must be less than 5MB.";
            return RedirectToReturnUrl(returnUrl);
        }

        try
        {
            // יצירת תיקיית uploads אם לא קיימת
            var uploadsFolder = Path.Combine(Directory.GetCurrentDirectory(), "wwwroot", "uploads");
            if (!Directory.Exists(uploadsFolder))
                Directory.CreateDirectory(uploadsFolder);

            // יצירת שם קובץ ייחודי: GUID + שם מקורי
            var uniqueFileName = $"{Guid.NewGuid()}_{Path.GetFileName(resumeFile.FileName)}";
            var filePath = Path.Combine(uploadsFolder, uniqueFileName);

            // שמירת הקובץ ל-Disk בצורה א-סינכרונית
            using (var fileStream = new FileStream(filePath, FileMode.Create))
            {
                await resumeFile.CopyToAsync(fileStream);
            }

            // URL יחסי לגישה לקובץ דרך הדפדפן
            resumeUrl = $"/uploads/{uniqueFileName}";
        }
        catch (Exception ex)
        {
            TempData["Error"] = $"An error occurred during file upload: {ex.Message}";
            return RedirectToReturnUrl(returnUrl);
        }
    }

    // שליחת בקשת מועמדות ל-API עם ה-URL של הקובץ
    ApiClient<bool> client = BuildClient<bool>("User", "ApplyToJob");
    client.AddParameter("userId", userId);
    client.AddParameter("jobId", jobId.ToString());
    if (resumeUrl != null)
        client.AddParameter("resumeUrl", resumeUrl);
    
    bool success = await client.PostAsync();
    TempData[success ? "Success" : "Error"] = success
        ? "Application submitted successfully!"
        : "Failed to apply. You may have already applied to this job.";

    return RedirectToReturnUrl(returnUrl);
}
```

**אלגוריתם שמירת הקובץ:**
1. בדיקה שתיקיית `wwwroot/uploads/` קיימת; אם לא – יצירתה.
2. יצירת שם קובץ ייחודי: `{GUID}_{OriginalFilename}`. ה-GUID מבטיח שאין התנגשות בין קבצים.
3. יצירת `FileStream` ל-Disk ו-`CopyToAsync` של נתוני הקובץ – a-סינכרוני.
4. שמירת ה-URL היחסי (`/uploads/uniquename.pdf`) כחלק מרשומת המועמדות.

# פרק 7: סיכום אישי ורפלקציה

## תיאור תהליך העבודה

הפרויקט התחיל עם לא מעט ספק עצמי: האם אוכל לפתח מערכת מלאה עם שרת API, לקוח MVC, מסד נתונים ומערכת צ'אט – בעצמי, בתוך כמה חודשים? לאחר תהליך ארוך של למידה, כישלונות, תיקונים ועוד כישלונות, ניתן לומר שהתשובה היא כן.

**שלב ראשון – תכנון:** הקדשתי שלושה שבועות לתכנון בלבד. ציירתי את ה-ERD, תרשימי ה-DFD ואת זרימת המסכים. זה שלב שניסיתי לדלג עליו בפרויקטים קודמים ושילמתי על כך במחיר יקר. הפעם, ההשקעה בתכנון חסכה זמן רב בשלבי הפיתוח.

**שלב שני – פיתוח שכבת הנתונים:** המאתגר בין כולם. בניית מחלקת `DBHelperOledb` והבנת מנגנון ה-OleDb לקחה זמן, אך ברגע שהיא עבדה עם Repository אחד, הרחבתה ל-11 Repositories הייתה פשוטה יחסית, בזכות תבנית ה-Repository.

**שלב שלישי – פיתוח ה-API:** פיתוח ה-Controllers היה שלב מרגש, כיוון שלראשונה ראיתי Swagger מחזיר תוצאות אמיתיות מה-API. בדיקת כל Endpoint בנפרד דרך ממשק ה-Swagger חסכה שעות של Debug.

**שלב רביעי – פיתוח הלקוח:** שלב זה היה הארוך ביותר. ניהול ה-Session, בניית ה-ApiClient הגנרי, ההתמודדות עם הגדרות Routing ועיצוב 15+ עמודי Razor לקחו חמישה שבועות.

## הצלחות וגאוות

**הצלחה מרכזית 1 – מחלקת `ApiClient<T>` הגנרית:** כאשר הצלחתי לבנות מחלקה גנרית אחת שמסוגלת לבצע קריאות GET ו-POST לכל endpoint ב-API ולהחזיר כל סוג אובייקט, חשתי גאווה אמיתית. זה פתרון אלגנטי שחסך מאות שורות קוד חוזר.

**הצלחה מרכזית 2 – מנגנון העלאת PDF:** מימוש מלא של העלאת קבצים עם ולידציה בשני שלבים (Client + Server), שמירה מאובטחת עם שמות GUID ייחודיים, ושילוב ב-URL במסד הנתונים – זה פיתוח שדרש מחקר עצמאי משמעותי.

**הצלחה מרכזית 3 – שאילתת UNION לצ'אט:** השאילתה שמוצאת את כל שותפי השיחה של משתמש (גם שולחים גם מקבלים) בשאילתה אחת אלגנטית עם UNION – זו הייתה "אהה!" רגע של הבנה אמיתית של יכולות SQL.

## אתגרים וקשיים

**קושי 1 – `OleDbException` לא ברורות:** Access מחזיר הודעות שגיאה לא תמיד ברורות. למשל, שגיאת "Unspecified error" שיכולה לנבוע מ-JOIN לא תקין, ממפתח זר שגוי, מסוג נתונים לא מתאים ועוד. פתרון: מידת סבלנות רבה, שימוש ב-Console.WriteLine לכל שלב, ובניית שאילתות פשוטות ואז הוספת מורכבות בהדרגה.

**קושי 2 – סינון Genre לפי GenreID ו-GenreTitle:** מסד הנתונים שמר `GenreID` כמספר שלם, אבל הלקוח שלח אותו לפעמים כ-String ולפעמים ישירות כ-GenreTitle. הפתרון היה סינון כפול בקוד C# שמשווה גם לפי `GenreID` וגם לפי `GenreTitle` עם `Trim()`.

**קושי 3 – Session נמחק בין Requests:** בתחילת הדרך, Session של המשתמש נמחק לאחר כל בקשה. הסיבה: שכחתי להוסיף `app.UseSession()` ב-`Program.cs`. פשוט לתיקון, אבל גרם ל-Debug של שעות שלמות.

## תהליך למידה ותובנות

הפרויקט לימד אותי דברים שלא ניתן ללמוד מספרי לימוד בלבד:
- **ה"ה" של HTTP:** הבנה עמוקה של ההבדל בין GET ל-POST, מהו Body, מהו Query String.
- **Async ב-.NET:** לא רק כיצד לכתוב `async/await`, אלא *מדוע* זה חשוב לביצועי שרת.
- **SQL מעשי ב-MS Access:** כולל ניואנסים כמו `CStr()`, Nested JOINs ו-UNION.
- **גיט ובקרת גרסאות:** שמירת כל גרסה ב-Git גילתה ערכה פעמים רבות כאשר נדרשתי לחזור לגרסה קודמת.

**כלים שאקח להמשך:**
- הגישה של "בניית Prototype קטן תחילה" לפני מימוש מלא.
- תבנית Repository Pattern וה-Unit of Work – שניים מהכלים הנפוצים ביותר בתעשייה.
- Swagger לבדיקת API – כלי שחוסך שעות של Debug.
- ה-Generic Class `ApiClient<T>` – ניתן להתאים אותו לכל פרויקט עתידי.

```csharp
using JobWebService.ORM;
using JobWebService.ORM.Repositories;
using JobModels;

namespace JobWebService
{
    public class Program
    {
        public static void Main(string[] args)
        {
            var builder = WebApplication.CreateBuilder(args);

            // רישום שירותי ה-ASP.NET Core MVC:
            builder.Services.AddControllers();
            
            // DBHelperOledb ו-ModelCreators כ-Singleton:
            // Singleton = מופע יחיד לכל אורך חיי האפליקציה
            builder.Services.AddSingleton<DBHelperOledb>();
            builder.Services.AddSingleton<ModelCreators>();

            // רישום Repository לכל ישות (Scoped = מופע חדש לכל בקשה HTTP):
            builder.Services.AddScoped<IRepository<User>, UserRepository>();
            builder.Services.AddScoped<IRepository<Job>, JobRepository>();
            builder.Services.AddScoped<IRepository<Genre>, GenreRepository>();
            builder.Services.AddScoped<IRepository<Review>, ReviewRepository>();
            builder.Services.AddScoped<IRepository<Notification>, NotificationRepository>();
            builder.Services.AddScoped<IRepository<Country>, CountryRepository>();
            builder.Services.AddScoped<IRepository<Education>, EducationRepository>();
            builder.Services.AddScoped<IRepository<EducationType>, EducationTypeRepository>();
            builder.Services.AddScoped<IRepository<UserType>, UserTypeRepository>();
            builder.Services.AddScoped<IRepository<ChatMessage>, ChatMessageRepository>();
            
            // Swagger: תיעוד ובדיקת API אוטומטיים
            builder.Services.AddEndpointsApiExplorer();
            builder.Services.AddSwaggerGen();

            var app = builder.Build();

            // Swagger זמין רק בסביבת פיתוח (Development)
            if (app.Environment.IsDevelopment())
            {
                app.UseSwagger();
                app.UseSwaggerUI();
            }

            app.UseAuthorization();
            app.MapControllers();
            app.Run();
        }
    }
}
```

```csharp
namespace JobWebApp
{
    public static class SessionHelper
    {
        private const string USER_ID   = "UserID";
        private const string USER_TYPE = "UserTypeID";
        private const string USER_NAME = "UserName";
        private const string FULL_NAME = "FullName";

        public static void SetUser(ISession session, string userId, int userTypeId,
                                    string userName, string fullName)
        {
            session.SetString(USER_ID,   userId);
            session.SetString(USER_TYPE, userTypeId.ToString());
            session.SetString(USER_NAME, userName);
            session.SetString(FULL_NAME, fullName);
        }

        public static string? GetUserID(ISession session)   => session.GetString(USER_ID);
        public static string? GetUserName(ISession session) => session.GetString(USER_NAME);
        public static string? GetFullName(ISession session) => session.GetString(FULL_NAME);

        public static int GetUserTypeID(ISession session)
        {
            int.TryParse(session.GetString(USER_TYPE), out int result);
            return result;
        }

        public static bool IsLoggedIn(ISession session) =>
            !string.IsNullOrEmpty(session.GetString(USER_ID));

        public static bool IsEmployee(ISession session) => GetUserTypeID(session) == 2;
        public static bool IsEmployer(ISession session) => GetUserTypeID(session) == 3;
        public static bool IsAdmin(ISession session)    => GetUserTypeID(session) == 4;

        public static void ClearUser(ISession session) => session.Clear();
    }
}
```
